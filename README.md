# Mark — Система маркировки

Десктоп-приложение для пищевого производства: печать этикеток на товары и учёт
статистики. Работает на Windows, написано на Python + PyQt5.

Возможности:
- формирование и печать этикеток на термопринтер (протокол TSPL, 203 dpi);
- данные о продуктах, сотрудниках и цехах берутся из Excel-файла;
- считывание веса с весов через COM-порт (RS-232 / RS-485) — кнопка «Взвесить»;
- модуль статистики: дашборды на экране и печать отчётов на A4;
- архив напечатанных этикеток с автоочисткой.

Продукт выпускается под брендом **LDN Tech**. © 2026 LDN Tech.

## Требования

- **Windows** 10/11 (x64).
- **Python 3.10+**.
- **PyQt5** — именно 5-я версия. **PyQt6 использовать нельзя:** целевой планшетный
  терминал на Intel Celeron J1900 не поддерживает AVX, и PyQt6 там не запускается.
- Термопринтер с поддержкой **TSPL** (целевой — 4B-2054L, 203 dpi) — нужен только
  для реальной печати, для разработки не обязателен.

## Быстрый старт (для разработки)

1. **Получить код.** Либо склонировать:
   ```
   git clone <URL-репозитория>
   ```
   либо на странице GitHub нажать **Code → Download ZIP** и распаковать.

2. **Открыть папку в VS Code** (File → Open Folder).

3. **Создать виртуальное окружение и установить зависимости** (в терминале VS Code,
   PowerShell):
   ```
   py -m venv .venv
   .\.venv\Scripts\Activate.ps1
   python -m pip install -r requirements.txt
   ```
   В VS Code выберите интерпретатор из `.venv` (Ctrl+Shift+P → «Python: Select
   Interpreter»).

4. **Запустить приложение:**
   ```
   python main.py
   ```

> Если PowerShell не разрешает активацию окружения, выполните один раз:
> `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`.

## Сборка .exe (PyInstaller)

В активированном окружении установите PyInstaller (`python -m pip install pyinstaller`)
и соберите:

```
py -m PyInstaller --noconfirm --clean --name Mark --icon=assets\mark_app.ico --version-file version_info.txt --onedir --noconsole --add-data "assets;assets" --add-data "data_sources;data_sources" main.py
```

Результат: `dist\Mark\Mark.exe` (со всеми файлами рядом).
Эквивалентно можно собрать из спека: `pyinstaller Mark.spec`.

## Сборка установщика (Inno Setup)

1. Установите **Inno Setup**.
2. Убедитесь, что приложение уже собрано в `dist\Mark\`.
3. Откройте `installer.iss` в Inno Setup и нажмите **F9** (Compile).

Результат: `installer\MarkSetup.exe`.
Пользователь получает только этот файл — исходники, git и прочая «кухня» до него
не доходят.

## Данные пользователя

Приложение хранит свои данные в профиле пользователя Windows:

```
%LOCALAPPDATA%\MirlisMark\
├── settings.json              — настройки
├── Готовые этикетки\          — архив напечатанных этикеток (хранение 365 дней)
├── ... журнал статистики
└── scale_link.json            — кэш найденного подключения весов
```

> Имя внутренней папки исторически `MirlisMark` — оно не видно пользователю.
> Менять его не нужно: при переименовании потеряются настройки и архив у уже
> установленных копий.

## Заметки для будущих разработчиков

- **Только PyQt5** (см. «Требования»). Синтаксис enum'ов — старого стиля
  (`Qt.AlignCenter`).
- **Excel** должен содержать ровно три листа: «продукт», «изготовил», «цех».
  Данные автоматически перечитываются раз в ~60 секунд.
- **Печать** идёт через виртуальный холст фиксированного размера (не от размеров
  окна) — это важно для одинакового результата на разных экранах/DPI.
- **Весы.** `scale_reader.py` сам перебирает порты, скорости и протоколы
  (tEnSo «Тензо-М», 6.43, потоковый ASCII / MT-SICS), принимает вес и кэширует
  рабочую комбинацию. Для проверки без реального прибора есть эмулятор — он не
  входит в репозиторий как часть приложения.
- **Два режима** интерфейса: ПК и планшет (различия в масштабировании под DPI).
